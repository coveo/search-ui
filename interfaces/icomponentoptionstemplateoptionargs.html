Interface IComponentOptionsTemplateOptionArgs

The IComponentOptionsTemplateOptionArgs interface describes the available parameters when building a template option.

Optional alias

alias: string | string[]

Inherited from IComponentOptions.alias

  • Defined in src/ui/Base/ComponentOptions.ts :102

Specifies an alias, or array of aliases, which can be used instead of the actual option name.

Note:

This can be useful to modify an option name without introducing a breaking change.

Optional attrName

attrName: string

Inherited from IComponentOptions.attrName

  • Defined in src/ui/Base/ComponentOptions.ts :93

Specifies a different markup name to use for an option, rather than the standard name (i.e., data- followed by the hyphened name of the option).

Note:

This should only be used for backward compatibility reasons.

Optional childSelector

childSelector: string
  • Defined in src/ui/Base/ComponentOptions.ts :225

Specifies the CSS selector the templates must match. The list of all matching, valid elements in the page is used as the template option value.

Default value is ., followed by the hyphened name of the template option being configured (e.g., .content-template, .result-template, .sub-result-template, .preview-template, etc.).

defaultFunction: function

Optional depend

depend: string

Inherited from IComponentOptions.depend

  • Defined in src/ui/Base/ComponentOptions.ts :117

Specifies the name of a boolean component option which must be true in order for this option to function properly.

Note:

This is mostly useful for the Coveo JavaScript Interface Editor.

Optional deprecated

deprecated: string

Inherited from IComponentOptions.deprecated

  • Defined in src/ui/Base/ComponentOptions.ts :129

Specifies a message that labels the option as deprecated. This message appears in the console upon initialization if the deprecated option is used in the page. Consequently, this message should explain as clearly as possible why the option is deprecated, and what now replaces it.

Note:

Deprecated options do not appear in the Coveo JavaScript Interface Editor.

Optional idAttr

idAttr: string
  • Defined in src/ui/Base/ComponentOptions.ts :234

Specifies the HTML id attribute the template must match. The corresponding template must be registered in the TemplateCache to be usable as the template option value.

If specified, this parameter takes precedence over childSelector.

Optional postProcessing

postProcessing: IComponentOptionsPostProcessing<Template>

Inherited from IComponentOptions.postProcessing

  • Defined in src/ui/Base/ComponentOptions.ts :83

Specifies a function that should allow a component option to further modify its own value once all other options of that component have been built.

Example:

By default, the id option of the Facet component uses a post processing function to set its value to that of the field option.

Optional required

required: boolean

Inherited from IComponentOptions.required

  • Defined in src/ui/Base/ComponentOptions.ts :72

Specifies whether it is necessary to explicitly specify a value for the option in order for the component to function properly.

Example:

The field option of the Facet component is required, since a facet cannot function properly without a field.

Optional section

section: string

Inherited from IComponentOptions.section

  • Defined in src/ui/Base/ComponentOptions.ts :107

Specifies a section name inside which the option should appear in the Coveo JavaScript Interface Editor.

Optional selectorAttr

selectorAttr: string
  • Defined in src/ui/Base/ComponentOptions.ts :216

Specifies the CSS selector the template must match. The first matching element in the page is used as the template option value, if this element is a valid template.

If specified, this parameter takes precedence over idAttr.
