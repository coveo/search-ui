Class Facet

This component displays a facet of the results for the current query. A facet consists of a list of values for a given field occurring in the results, ordered using a configurable criteria.
The list of values is obtained using a IGroupByRequest operation performed at the same time as the main query.
The Facet component allows the user to drill down inside results by restricting to certain field values. It also allows filtering out values, and can provide a search box to look for specific values inside larger sets.
It is probably the most complex component in the Coveo UI, and as such allows many different options.

Specifies the sort criteria options you want to be displayed in the facet settings menu.

Possible values are : occurrences, score, alphaAscending, alphaDescending, computedfieldascending, computedfielddescending, custom The default value is occurrences,score,alphaAscending,alphaDescending.

computedField

computedField: string
  • Defined in src/ui/Facet/Facet.ts :275

Specifies the name of a field on which an aggregate operation should be executed for all distinct values of the facet's field.
The result of the operation is displayed along with the number of occurrences for each value.
You can use this option to compute the sum of a field (like a money amount) for each facet value that is listed.
Works in conjunction with Facet.options.computedFieldOperation , Facet.options.computedFieldFormat, Facet.options.computedFieldCaption

computedFieldCaption

computedFieldCaption: string
  • Defined in src/ui/Facet/Facet.ts :305

Specifies what will be the caption of the Facet.options.computedField in the settings menu for sort.
For example, the value Money will be displayed as Money Ascending for computed field ascending.
The default value is Computed Field.

computedFieldFormat

computedFieldFormat: string
  • Defined in src/ui/Facet/Facet.ts :299

Specifies how to format the values resulting from a Facet.options.computedFieldOperation.
The formats available are defined by the Globalize library. The most common used formats are:

  • c0 - Formats the value as a currency.
  • n0 - Formats the value as an integer.
  • n2 - Formats the value as a floating point with 2 decimal digits.
See : Globalize for more informations.
Default value is 'c0

computedFieldOperation

computedFieldOperation: string
  • Defined in src/ui/Facet/Facet.ts :287

Specifies the type of aggregate operation to use on the Facet.options.computedField.
The available values are:

  • sum - Computes the sum of the computed field values.
  • average - Computes the average of the computed field values.
  • minimum - Finds the minimum value of the computed field values.
  • maximum - Finds the maximum value of the computed field values.

The default value is sum.

preservePosition

preservePosition: boolean
  • Defined in src/ui/Facet/Facet.ts :317

Specifies whether the facet should remain stable in its current position in the viewport while the mouse is over it.
Whenever values are selected in a facet, a new query is automatically performed. This new query might cause other elements in the page to be resized (typically other facets above the one being used).
When this option is enabled, the facet will adjust the scroll amount of the page to ensure that it does not move relative to the mouse when the results are updated.
In some cases, the facet will also add margin to the scrollContainer, if scrolling alone is not enough to preserve position.
This is the option that will add a div 'coveo-topSpace' / 'coveo-bottomSpace` around the facet container. The default value is true.

scrollContainer: HTMLElement

Specifies the HTML element (through a CSS selector) whose scroll amount the facet should adjust to preserve its position Facet.options.preservePosition when results are updated.
The default value is document.body.
