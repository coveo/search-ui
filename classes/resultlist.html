Class ResultList

This component is responsible for displaying the results of the current query using one or more result templates.
It supports many additional features such as infinite scrolling.

Component Options

autoSelectFieldsToInclude

autoSelectFieldsToInclude: boolean
  • Defined in src/ui/ResultList/ResultList.ts :116

Specifies that the result list should scan it's template and discover which field it will need to render every results.
This is to ensure that fields that are not needed for the UI to function are not sent by the search API.
Default value is false.
NB: Many interface created by the interface editor will actually explicitly set this option to true.

enableInfiniteScroll

enableInfiniteScroll: boolean
  • Defined in src/ui/ResultList/ResultList.ts :85

Specifies whether the ResultList automatically retrieves an additional page of results and appends them to those already being displayed whenever the user scrolls to the end of the infiniteScrollContainer.
The waitAnimation will be displayed while additional results are fetched.
Default value is false

enableInfiniteScrollWaitingAnimation

enableInfiniteScrollWaitingAnimation: boolean
  • Defined in src/ui/ResultList/ResultList.ts :101

Specifies if the wait animation should be displayed when a query is being performed using infinite scroll.
Default value is true

fieldsToInclude

fieldsToInclude: Array<string>
  • Defined in src/ui/ResultList/ResultList.ts :109

Specifies a list of fields to include in the query.
This is to ensure that fields that are not needed for the UI to function are not sent by the search API.
By default, this list is empty.
Note that this option has an interaction with autoSelectFieldsToInclude

infiniteScrollContainer

infiniteScrollContainer: HTMLElement
  • Defined in src/ui/ResultList/ResultList.ts :96

When infinite scrolling is enabled, specifies the element whose scrolling is monitored to trigger fetching of additional results.
By default, the framework will try to find the first scrolling parent it encounter, starting from the ResultList itself
This also means that if it encounter no parent that are scrollable (in css this means having overflow-y: scroll), then the window itself will be the scroll container

infiniteScrollPageSize

infiniteScrollPageSize: number
  • Defined in src/ui/ResultList/ResultList.ts :90

When infiniteScroll is enabled, specifies the number of additional results that are fetched when the user scrolls to the bottom of the infiniteScrollContainer.
Default value is 10

resultContainer

resultContainer: HTMLElement
  • Defined in src/ui/ResultList/ResultList.ts :56

Specifies the element within which the rendered templates for results are inserted.
The content of this element is cleared when a new query is performed. If this option is not specified, a <div> element will by dynamically created in javascript and appended to the result list and used as a result container.
You can change the container by specifying it's selector: Eg data-result-container-selector="#someCssSelector"

waitAnimation

waitAnimation: string
  • Defined in src/ui/ResultList/ResultList.ts :72

Specifies the type of animation to display while waiting for a new query to finish executing.
Possible values are :
'fade' : Fades out the currently displayed results while the query is executing.
'spinner' : Shows a spinning animation while the query is executing.
'none' : Use no animation during queries.
Default value is 'none'

waitAnimationContainer

waitAnimationContainer: HTMLElement
  • Defined in src/ui/ResultList/ResultList.ts :79

Specifies the element inside which an animation is displayed while waiting for a new query to finish executing.
You can change this by specifying a css selector.
Eg : data-wait-animation-container-selector="#someCssSelector" By default, the animation appears in the the resultContainer.
